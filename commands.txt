Основные команды:


git init: Инициализирует новый пустой Git репозиторий или превращает существующий каталог в Git репозиторий.  Используется для начала отслеживания изменений в проекте.

git clone <URL>:  Копирует (клонирует) репозиторий, расположенный по указанному URL, в локальную директорию.  Используется для получения копии существующего репозитория.

git add <file(s)>: Добавляет изменения из рабочей директории в индекс (staging area).  Подготавливает файлы к коммиту.  git add . добавит все измененные файлы.

git commit -m "<message>":  Фиксирует изменения, находящиеся в индексе, в локальный репозиторий с указанным сообщением.  Сообщение должно быть кратким и описательным.

git status: Показывает состояние рабочей директории и индекса.  Показывает, какие файлы отслеживаются, изменены, добавлены в индекс и т.д.

git log: Отображает историю коммитов текущей ветки. Показывает автора, дату и сообщение коммита.


Работа с ветками:


git branch:  Показывает список локальных веток.  git branch <name> создает новую ветку с именем <name>.

git checkout <branch>: Переключает рабочую директорию на указанную ветку.

git merge <branch>:  Сливает указанную ветку в текущую ветку.

git branch -d <branch>:  Удаляет указанную ветку (если она была слита).  Используйте -D для принудительного удаления ветки.


Работа с удаленным репозиторием:


git remote add origin <URL>:  Устанавливает связь между локальным репозиторием и удаленным репозиторием (обычно origin).

git push origin <branch>:  Отправляет локальные коммиты в удаленный репозиторий на указанную ветку.

git pull origin <branch>:  Получает изменения из удаленного репозитория и сливает их с текущей веткой.  Важно делать git pull перед git push для избежания конфликтов.

git fetch: Скачивает объекты и ссылки из другого репозитория. в отличие от pull, fetch не делает слияние изменений.
Отмена изменений:

git reset --hard <commit>:  Возвращает репозиторий к указанному коммиту, полностью отменяя все последующие изменения.  ОПАСНО!  Потеря данных!

git revert <commit>: Создает новый коммит, который отменяет изменения, внесенные в указанном коммите.  Более безопасный способ отмены изменений, так как сохраняет историю.


Прочее:


git diff: Показывает изменения между рабочей директорией и индексом или между двумя коммитами.

.gitignore:  Файл в корне репозитория, указывающий, какие файлы и папки Git должен игнорировать.
